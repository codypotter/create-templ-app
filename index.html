create-templ-app

Go + Gin + templ + Tailwind CSS v4 + htmx + Alpine.js, scaffolded with a single command.

Usage

go run github.com/codypotter/create-templ-app/cmd/create-templ-app@latest github.com/you/your-app
cd your-app
make tidy    # go mod tidy && npm install
make dev

This copies the repo itself into ./your-app and rewrites the module path throughout. You end up with a normal, standalone Go module — no dependency on this repo, no generator lock-in to remove later.

Why this stack

Server-rendered HTML with htmx and Alpine.js handling the interactive slivers, instead of a client-side framework. templ compiles views to typed Go rather than parsing templates at runtime, and Tailwind/esbuild are wired up with content-hashed, cacheable output from the start. The goal is to skip the SPA tax — hydration bugs, client-side state sprawl, a build toolchain that reinvents itself every year — for the large majority of apps that don't actually need it.

Stack

  • Go + Gin — HTTP server, server-side rendering
  • templ — HTML components compiled to Go
  • Tailwind CSS v4 — compiled via @tailwindcss/cli
  • Alpine.js + htmx — bundled via esbuild
