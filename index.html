create-templ-app

A minimal Go web stack, scaffolded with a single command.

  • Go
  • templ
  • Tailwind CSS v4
  • htmx
  • Alpine.js

Usage

go run github.com/codypotter/create-templ-app/cmd/create-templ-app@latest github.com/you/your-app
cd your-app
make tidy    # go mod tidy && npm install
make dev

This copies the repo itself into ./your-app and rewrites the module path throughout. You end up with a normal, standalone Go module — no dependency on this repo, no generator lock-in to remove later.

Why this stack

Server-rendered HTML with htmx and Alpine.js handling interactivity, instead of a client-side framework. templ compiles views to typed Go rather than parsing templates at runtime, and Tailwind/esbuild are wired up with content-hashed, cacheable output. The goal is to skip the SPA tax for the large majority of apps that don't need it.

Built to be changed

Router

Gin only shows up in one ~40-line file. Views render to any io.Writer, so nothing about templ or htmx ties you to it — swap in chi, echo, or stdlib net/http whenever you want.

Frontend build

build/build.js is a plain esbuild script you can read start to finish in a minute. Add loaders, TypeScript, more entry points — no framework CLI standing between you and the config.

Dependencies

Three direct Go dependencies for the whole server. Easy to audit, fast to build, nothing to wait on for security patches.

Deploy

Go compiles to a single binary; assets are pre-built and content-hashed. Shipping is "copy a binary and a folder" — no Node runtime in production.
